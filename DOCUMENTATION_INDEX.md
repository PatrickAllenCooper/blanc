## Documentation Standards

### All Documents Should Have
1. **Title** - Clear, descriptive
2. **Date** - Creation/update date
3. **Status** - Complete, draft, archived, etc.
4. **Purpose** - Why this document exists
5. **Sections** - Clear organization with headings

### Markdown Guidelines
- Use `#` for main title (H1)
- Use `##` for sections (H2)
- Use `###` for subsections (H3)
- Use code blocks with language tags
- Use lists for items
- Use bold/italic sparingly
- Include horizontal rules `---` to separate major sections

## Document Lifecycle

### Creation
1. Create document with header (title, date, status, purpose)
2. Write content
3. Add to git: `git add <file>`
4. Commit: `git commit -m "Add <description>"`
5. Update this index if it's a major document

### Updates
1. Update date in header
2. Make changes
3. Commit with descriptive message
4. Update STATUS.md if status changed

### Archival
1. Move to appropriate `docs/` subdirectory
2. Add "ARCHIVED" to title
3. Update index to mark as historical
4. Keep in git for history
